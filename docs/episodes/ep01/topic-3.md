# 冲突升级：产品经理用 AI 出了设计稿和代码，设计和前端怎么看？

## 议题背景

这不是假设，这是正在发生的事。

2025-2026 年，AI 工具链已经成熟到一个产品经理可以：用 ChatGPT 写 PRD → 用 v0.dev 生成 UI → 用 Cursor 写出可运行的代码 → 用 AI 测试工具跑基础用例。全程不需要设计师和前端工程师介入。

Maor Shlomo 一个人用 AI 开发 Base44 平台，6 个月内达到 25 万用户，无联合创始人、无员工。Pieter Levels 零员工运营十几个产品，月收入超过 30 万美元。

当"越界"变成常态，岗位边界还有意义吗？

## 📊 数据速览

| 案例 | 发生了什么 | 结果 |
|------|-----------|------|
| **Base44 (Maor Shlomo)** | 一人用 AI 开发应用平台 | 6 个月 25 万用户，零员工 |
| **Pieter Levels** | 零员工运营 12+ 产品 | 月收入 $300K+ |
| **Shopify (2025.4)** | CEO 备忘录：先证明 AI 做不了再招人 | 全员必须掌握 AI |
| **Klarna (2025.9)** | AI 替代客服后质量下降 | 部分员工被调回客服岗 |
| **AI 工具采用率** | 92% 开发者使用 AI | 但仅 34% 感到真正提效 |

:::danger 灵魂拷问
如果产品经理 3 天就能做出一个"能用"的产品，设计师和前端还有 2 周的时间窗口吗？
:::
